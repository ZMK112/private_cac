在隔离的 Docker 容器中运行 Claude Code，支持 sing-box TUN 透明代理、cac 身份保护，以及供子容器工作流使用的宿主 Docker 代理通道。

详细设置说明请参见 [Docker 模式指南](/guides/docker-mode)。

执行 `bash install.sh --local --yes` 后，Docker 资源会自动链接到 `~/.cac/docker`，因此你可以在任意目录直接运行 `cac docker ...`。执行 `cac docker start` 时所在的当前目录会挂载到容器内的 `/workspace`。

## 命令

| 命令 | 说明 |
|:---|:---|
| `cac docker setup` | 配置代理和网络（交互式） |
| `cac docker create` | 构建或拉取镜像 |
| `cac docker start` | 启动容器 |
| `cac docker stop` | 停止容器 |
| `cac docker restart` | 重启容器 |
| `cac docker enter` | 进入容器 Shell |
| `cac docker check` | 网络 + 身份诊断 |
| `cac docker port <port>` | 将端口转发到本地主机 |
| `cac docker status` | 显示容器状态 |
| `cac docker logs` | 查看容器日志 |
| `cac docker destroy` | 移除容器和网络 |

## 代理格式

Docker 模式除了支持紧凑格式外，还支持分享链接：

```
ip:port:user:pass          # 紧凑格式（SOCKS5）
host:port                  # 使用主机名的紧凑格式（SOCKS5）
ss://...                   # Shadowsocks
vmess://...                # VMess
vless://...                # VLESS
trojan://...               # Trojan
```

## 网络模式

| 模式 | 适用场景 | 方式 |
|:---|:---|:---|
| 远程（macvlan） | 使用 Docker Engine 的 Linux 服务器 | 容器在局域网上获得独立 IP，完全绕过主机网络栈 |
| 本地（bridge） | 使用 Docker Desktop 的笔记本电脑 | 容器使用 Docker 桥接网络，代理流量加密传输 |

模式根据你的 Docker 安装方式自动检测。

## 说明

- `cac docker setup` 会把配置写入当前使用的 Docker 资源目录：本地安装后通常是 `~/.cac/docker/.env`，在源码树中运行时则是 `docker/.env`。
- `cac docker enter` 会进入主容器的 shell，里面已经准备好 `claude`、`cac` 和 `docker`。
- `cac docker check` 只校验主容器的 TUN 代理路径。子容器默认走宿主 Docker 网络，除非你显式注入代理环境变量。
