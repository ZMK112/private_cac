在隔离的 Docker 容器中运行 Claude Code，支持 sing-box TUN 透明代理、cac 身份保护，以及供子容器工作流使用的宿主 Docker 代理通道。

详细设置说明请参见 [Docker 模式指南](/guides/docker-mode)。

执行 `bash install.sh --local --yes` 后，Docker 资源会自动链接到 `~/.cac/docker`，因此你可以在任意目录直接运行 `cac docker ...`。执行 `cac docker start` 时所在的当前目录会挂载到容器内的 `/workspace`。

## 命令

| 命令 | 说明 |
|:---|:---|
| `cac docker setup` | 配置代理和网络（交互式） |
| `cac docker create` | 构建或拉取镜像 |
| `cac docker rebuild` | 强制重建本地镜像；如果容器已存在则自动重建容器 |
| `cac docker start` | 启动容器 |
| `cac docker stop` | 停止容器 |
| `cac docker restart` | 重启容器 |
| `cac docker enter` | 进入容器 Shell |
| `cac docker check` | 网络 + 身份诊断 |
| `cac docker port <port>` | 将端口转发到本地主机 |
| `cac docker status` | 显示容器状态 |
| `cac docker logs` | 查看容器日志 |
| `cac docker destroy` | 移除容器和网络 |

## Web UI

Docker 模式现在在主容器内包含一个可通过浏览器访问的 CloudCLI Web UI。

- 默认发布到 `0.0.0.0:${CAC_HOST_WEB_PORT:-3001}`
- 绑定地址可通过 `CAC_HOST_WEB_BIND` 配置：
  - `0.0.0.0` 表示允许局域网访问
  - `127.0.0.1` 表示仅本机访问
- 如果 `CAC_HOST_WEB_PORT` 已被占用，`cac docker start` / `cac docker rebuild` 会自动顺延到下一个空闲宿主端口，并把新端口写回 `docker/.env`
- 运行在和 `cac docker enter` 相同的主保护容器里
- 目标是直接使用 Docker 模式里的当前活动 `cac` 环境
- 默认以免登录的平台模式运行，正常情况下打开页面会直接进入应用，而不是停在 CloudCLI 登录页
- 安全提示：默认 `CAC_HOST_WEB_BIND=0.0.0.0` 时，这个免登录 Web UI 会直接暴露给局域网。只应在受信任网络使用，或者改成 `CAC_HOST_WEB_BIND=127.0.0.1`。

在宿主机上打开：

```bash
http://127.0.0.1:3001
```

如果你要改回仅本机访问，在 `docker/.env` 里设置：

```env
CAC_HOST_WEB_BIND=127.0.0.1
CAC_HOST_WEB_PORT=3001
```

说明：

- Web UI 不是一个放宽隔离的独立 sidecar，而是主保护容器的一部分
- 隐私/网络正确性仍以 `cac docker check` 为准
- 如果 Docker 模式保护检查回归，即使 Web UI 能打开，也不能视为有效
- 子容器现在统一通过一个带认证的 `proxy-bridge` sidecar 出网：它会把上游 `PROXY_URI` 转成标准 HTTP/SOCKS 代理地址，因此 `ss://`、`vmess://`、`vless://`、`trojan://` 这类分享链接协议也能被子容器工作流继承。

## 代理格式

Docker 模式除了支持紧凑格式外，还支持分享链接：

```
ip:port:user:pass          # 紧凑格式（SOCKS5）
host:port                  # 使用主机名的紧凑格式（SOCKS5）
socks5h://host:port        # 显式 SOCKS5
http://host:port           # 显式 HTTP 代理
ss://...                   # Shadowsocks
vmess://...                # VMess
vless://...                # VLESS
trojan://...               # Trojan
```

在 Docker 模式里，裸 `host:port` 会按 SOCKS5 解释；如果你的代理是纯 HTTP，请明确写成 `http://...`。

## 网络模式

| 模式 | 适用场景 | 方式 |
|:---|:---|:---|
| 远程（macvlan） | 使用 Docker Engine 的 Linux 服务器 | 容器在局域网上获得独立 IP，完全绕过主机网络栈 |
| 本地（bridge） | 使用 Docker Desktop 的笔记本电脑 | 容器使用 Docker 桥接网络，代理流量加密传输 |

模式根据你的 Docker 安装方式自动检测。

## 说明

- `cac docker setup` 会把配置写入当前使用的 Docker 资源目录：本地安装后通常是 `~/.cac/docker/.env`，在源码树中运行时则是 `docker/.env`。
- Docker 模式会把 `/root` 和 `/home` 持久化到 `CAC_DATA`（默认就是当前 Docker 资源目录下的 `docker/data`）。重建镜像、重建容器或重新安装默认都会保留 Claude 登录状态、记忆和历史 session；只有你手动删除这个数据目录时才会丢失。
- `cac docker status` 现在会同时显示原始 `CAC_DATA` 设置和解析后的绝对路径，方便你在做任何破坏性清理前先备份。
- `setup` 向导在正常情况下只会询问代理，部署模式会自动检测，其他启动所需变量会自动写入默认值；只有远程模式的 macvlan 自动探测失败时，才会继续询问网络参数。
- 在本地 Docker Desktop 模式下，如果你输入的是 `127.0.0.1:port` 或 `localhost:port`，`setup` 会自动改写成 `host.docker.internal:port`。
- `cac docker enter` 会进入主容器的 shell，里面已经准备好 `claude`、`cac` 和 `docker`。
- `cac docker check` 现在会校验主容器的 TUN 代理路径、检测出口 IP 地区并在中国出口时直接报错，同时在启用 SSH 时检查宿主机侧发布的 SSH 端口。
- Docker 模式现在会默认把 SSH 发布到 `0.0.0.0:${CAC_HOST_SSH_PORT:-2222}`，因此局域网客户端可以直接用 `ssh -p 2222 cherny@<宿主机IP>` 连接；默认密码是 `cherny`，除非你显式改了 `CAC_SSH_PASSWORD`。
- 如果配置的 SSH 端口已经被占用，Docker 模式也会自动顺延到下一个空闲端口，并把结果持久化到 `docker/.env`。
- 如果要改回仅本机访问，请设置 `CAC_HOST_SSH_BIND=127.0.0.1` 后重启栈。
- Docker 模式现在会默认把 Web UI 发布到 `0.0.0.0:${CAC_HOST_WEB_PORT:-3001}`。
- 如果要改回仅本机访问，请设置 `CAC_HOST_WEB_BIND=127.0.0.1` 后重启栈。
- 如果你保留这两个默认值（`CAC_HOST_SSH_BIND=0.0.0.0` 和 `CAC_HOST_WEB_BIND=0.0.0.0`），要意识到局域网内其他设备既能访问密码登录 SSH，也能直接访问免登录 Web UI。
- 如果你想快速改回“仅本机访问”，可以直接在 `docker/.env` 或 `~/.cac/docker/.env` 里写：

```env
CAC_HOST_SSH_BIND=127.0.0.1
CAC_HOST_WEB_BIND=127.0.0.1
```

- `BUILD_PROXY` 是可选的，仅用于镜像构建阶段的下载；如果 `cac docker create` / `cac docker rebuild` 在下载 `claude.ai/install.sh` 之类的外部资源时失败，再额外设置一个标准 `http://...` 或 `socks5h://...` 的 `BUILD_PROXY` 后重试。
