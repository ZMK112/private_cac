## 命名规范

Docker 模式统一使用 `boris-` 前缀来替代 `cac`，所以主容器叫 `boris-main`，代理 sidecar 是 `boris-gateway`，设备主机名格式是 `boris-<short-uuid>`，工作区挂载在 `/workspace`。

模板文件 `docker/templates/cherny.*` 中冻结了 `TZ=Asia/Tokyo` 和 `LANG=ja_JP.UTF-8`，并对 `env`、prompt、telemetry` 的内容做了统一，避免将真实环境/设备信息汇报给 Anthropic。

## 构建与启动

```bash
./cac docker create
./cac docker start
```

结果是本地镜像和代理镜像都构建完成，`boris-gateway` 与 `boris-main` 成功运行，`/workspace` 绑到宿主当前目录。

## 检查结果

```text
Network
[TUN  ] ok   tun0 up (1ms)
[DNS  ] FAIL DNS resolution failed — check DNS_SERVER
[LEAK ] ok   DNS → 172.19.0.2 (TUN gateway, no leak) (5ms)
```

这个检查在 DNS 无法通过 `host.docker.internal:17890` 的 SOCKS5 代理时被终止（exit 137），因此后续的 TCP/HTTP/identity 信息没有打印出来。

### 代理建议

在宿主机上启动一个可访问的 SOCKS5 代理，再重新运行检查。例如：

```bash
cd docker
nohup python3 -u dev-socks5.py --host 0.0.0.0 --port 17890 >/tmp/dev-socks5.log 2>&1 &
```

如果你在 sandbox 环境里，如果 `setsid`/`nice` 不可用，`/tmp/dev-socks5.log` 会保持为空，请在真实的 macOS 主机上运行上述命令。

## 诊断与隐私说明

- 根据 ByteDance 提供的审计报告，Claude Code 会同时向 Datadog、BigQuery、GrowthBook 等渠道上报设备 ID、邮箱、系统指纹、GitHub metadata、`cch` attestation 等。Docker 模式在内层使用 `cherny.*` 模板彻底重写这些字段。
- docker-wrapper 会注入 `CAC_CHILD_CONTAINER_PROXY_URL` 等变量到 Claude 创建的子容器，配合 TUN 网络实现 CC Gateway 式的 fail-closed 防护。
- 将这一页作为审计记录保存下来，后续可以复用相同的命名、代理与检查步骤进行复验。
