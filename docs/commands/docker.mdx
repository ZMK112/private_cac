Run Claude Code in an isolated Docker container with sing-box TUN transparent proxy, cac identity protection, and a host-Docker proxy path for child-container workflows.

See the [Docker mode guide](/guides/docker-mode) for detailed setup instructions.

After `bash install.sh --local --yes`, Docker assets are linked into `~/.cac/docker`, so `cac docker ...` works from any directory. The directory where you run `cac docker start` is bind-mounted to `/workspace` inside the container.

## Commands

| Command | Description |
|:---|:---|
| `cac docker setup` | Configure proxy and network (interactive) |
| `cac docker create` | Build or pull the image |
| `cac docker rebuild` | Force-rebuild the local image and recreate the container if it already exists |
| `cac docker start` | Start the container |
| `cac docker stop` | Stop the container |
| `cac docker restart` | Restart the container |
| `cac docker enter` | Shell into the container |
| `cac docker check` | Network + identity diagnostics |
| `cac docker port <port>` | Forward a port to localhost |
| `cac docker status` | Show container status |
| `cac docker logs` | View container logs |
| `cac docker destroy` | Remove container and network |

## Web UI

Docker mode now includes a browser-accessible CloudCLI Web UI in the main container.

- Published by default on `0.0.0.0:${CAC_HOST_WEB_PORT:-3001}`
- Bind address is configurable with `CAC_HOST_WEB_BIND`:
  - `0.0.0.0` allows LAN access
  - `127.0.0.1` keeps it localhost-only
- If `CAC_HOST_WEB_PORT` is already occupied, `cac docker start` / `cac docker rebuild` automatically advance to the next free host port and write it back to `docker/.env`
- Backed by the same protected main container runtime used for `cac docker enter`
- Intended to use the current active `cac` env inside Docker mode
- Runs in no-login platform mode by default for Docker mode, so opening the page should go straight into the app instead of stopping on the CloudCLI login form

Open on the host:

```bash
http://127.0.0.1:3001
```

For localhost-only access, set in `docker/.env`:

```env
CAC_HOST_WEB_BIND=127.0.0.1
CAC_HOST_WEB_PORT=3001
```

Notes:

- The Web UI is part of the main protected container, not a separate relaxed sidecar
- `cac docker check` remains the primary gate for privacy/network correctness
- The Web UI must not be treated as valid if Docker-mode protection checks regress

## Proxy formats

Docker mode supports share links in addition to compact format:

```
ip:port:user:pass          # compact (SOCKS5)
host:port                  # compact (SOCKS5 hostname)
socks5h://host:port        # explicit SOCKS5
http://host:port           # explicit HTTP proxy
ss://...                   # Shadowsocks
vmess://...                # VMess
vless://...                # VLESS
trojan://...               # Trojan
```

Bare `host:port` is treated as SOCKS5 in Docker mode. If your proxy is HTTP-only, enter it with `http://`.

## Network modes

| Mode | When | How |
|:---|:---|:---|
| Remote (macvlan) | Linux server with Docker Engine | Container gets its own IP on the LAN, completely bypasses host network stack |
| Local (bridge) | Laptop with Docker Desktop | Container uses the compose bridge network, proxy traffic is encrypted |

Mode is auto-detected based on your Docker installation.

## Notes

- `cac docker setup` writes config into the active Docker resource directory: usually `~/.cac/docker/.env` after a local install, or `docker/.env` when running from the source tree.
- The setup wizard normally prompts only for the proxy. Deploy mode is auto-detected, and other required values are written with defaults automatically. In remote mode, it only asks for macvlan fields if auto-detection fails.
- In local Docker Desktop mode, entering `127.0.0.1:port` or `localhost:port` for the proxy is rewritten to `host.docker.internal:port` automatically.
- `cac docker enter` opens a shell in the main container. `claude`, `cac`, and `docker` are already on `PATH`.
- `cac docker check` validates the main container's TUN path, rejects China exit-IP geolocation, and verifies the published SSH endpoint when SSH is enabled.
- Docker mode now publishes SSH on `0.0.0.0:${CAC_HOST_SSH_PORT:-2222}` by default, so LAN clients can connect with `ssh -p 2222 cherny@<host-ip>` using the default password `cherny` unless you override `CAC_SSH_PASSWORD`.
- If the configured SSH port is already occupied, Docker mode automatically advances to the next free port and persists that port in `docker/.env`.
- To keep SSH localhost-only, set `CAC_HOST_SSH_BIND=127.0.0.1` and restart the stack.
- Docker mode now publishes the Web UI on `0.0.0.0:${CAC_HOST_WEB_PORT:-3001}` by default.
- To keep the Web UI localhost-only, set `CAC_HOST_WEB_BIND=127.0.0.1` and restart the stack.
