## v1.4.4-zmk.12-web.1
<sub>2026-04-07</sub>

**Separate Web-enabled variant release**

- Added a separate Web-enabled Docker-mode release track so it can evolve independently from the traditional Docker image/tag line.
- Added a browser-accessible CloudCLI UI inside the main protected `cac docker` container instead of using a separate relaxed Web sidecar.
- Added `s6-overlay` supervision for Docker-mode Web services and kept `Xvfb` + CloudCLI under the runtime-user model.
- Added `Chromium`, `Playwright`, and common terminal/dev tools such as `vim`, `tmux`, `less`, and `bat` to the Docker image.
- Added CloudCLI vendored-source tracking plus a local patch log for future upstream rebases.
- Added a `cac` path-compatibility layer so CloudCLI uses the active Docker-mode Claude config/session roots instead of assuming a bare `~/.claude`.
- Fixed Docker-mode active profile proxy drift by refreshing the active env `proxy` file from the current `PROXY_URI` on container start.
- Made the Docker-mode Web UI reachable from the host and LAN by default, with configurable bind addresses for both Web and SSH.
- Fixed the CloudCLI shell Disconnect behavior so a user-initiated disconnect no longer immediately auto-reconnects.
- Extended automated Docker validation to include Web UI readiness while preserving the original privacy/network/fail-closed gates.

## Unreleased
<sub>2026-04-01</sub>

**Docker runtime fixes**

- Added passwordless `sudo` for the default non-root Docker persona user so manual package installs inside `cac docker` containers no longer require dropping back to a root shell.
- Added a default Docker-mode utility baseline for data/dev workflows: `jq`, `ripgrep`, `fd`, `tree`, `sqlite3`, `zip`, and `unzip`.
- Added host-to-container SSH access in Docker mode: `sshd` now starts inside the main container, publishes a localhost-only SSH endpoint by default, and enables password login for the runtime user.
- Kept Docker-mode SSH compatible with later manual public-key login by leaving `PubkeyAuthentication` enabled, while still avoiding any host `~/.ssh` mount or automatic key import.
- Tightened `cac docker check` so it now fails when the container exit IP geolocates to China, and verifies the Docker-mode SSH path when SSH is enabled.
- Fixed `docker/entrypoint.sh` so Docker-mode containers no longer restart immediately after profile creation when the Chorny identity template is applied.
- Fixed local `cac docker port` forwarding by switching Docker Desktop/local mode to a helper-container publish model instead of relying on direct host access to the container bridge IP.
- Fixed `cac docker check` so it captures `cac-check` output from inside the container without depending on a fragile attached exec path.
- Hardened Docker-mode persona handling so the main process now runs as a non-root fake user by default, with stable `uid`/`gid`/`home`/`shell` surfaces in both shell and Node.js views.
- Added local container-trace masking for Docker mode: removes `/.dockerenv`/`/run/.containerenv`, sanitizes common `cgroup`/`mountinfo` reads, and extends `cac-check` to validate workspace writability plus hidden container markers.
- Made `cac docker create` / `cac docker start` more validation-friendly by avoiding unnecessary main-image rebuilds when the local image already exists.
- Fixed local-repo reinstall flow so `bash install.sh --local` now refreshes an existing Docker stack with the newly installed code, and added `CAC_DOCKER_REBUILD=1` to force a main-image rebuild when needed.
- Fixed release/source-archive installs so `install.sh` now recognizes an extracted source archive as a valid local checkout even when generated top-level runtime files are absent, and added a versioned portable release package asset.
- Fixed installed `cac docker` resolution so unrelated `docker/` directories under the current working directory no longer override the linked `~/.cac/docker` resources, and local Docker-mode setup now rewrites `127.0.0.1` / `localhost` proxy inputs to `host.docker.internal`.
- Added `cac docker rebuild` to force a local Docker image rebuild explicitly, without relying on `CAC_DOCKER_REBUILD=1`.
- Updated `cac docker rebuild` so it now recreates an existing container after rebuilding, ensuring the rebuilt image actually takes effect.
- Updated `cac docker setup` so derived child-proxy settings refresh when the main proxy changes, and clarified that bare `host:port` inputs are treated as SOCKS5 unless `http://` is specified.
- Fixed Docker sidecar socket mounting to follow the active Docker context's Unix socket instead of assuming `/var/run/docker.sock`, improving compatibility on machines where the CLI talks to Docker through `~/.docker/run/docker.sock`.
- Updated `cac docker start` and `cac docker rebuild` so they only report success after the main container has finished initialization and PID 1 has actually dropped to the runtime user.
- Tightened `bash install.sh --local` Docker auto-refresh: it now only refreshes an already running stack, and skips automatic startup on machines where Docker mode has not been reconfigured yet.
- Added explicit progress messages during `cac docker start` / `cac docker rebuild` readiness waits so it is clear whether startup is waiting on the container state, the in-container Docker API, or PID 1 privilege drop.
- Updated the Docker CLI bundled inside the main container so it can talk to newer host Docker daemons whose minimum API version is higher than Debian's old `docker.io` client supports.
- When Docker startup readiness times out, `cac docker start` / `cac docker rebuild` now stop the stack automatically instead of leaving a half-started container behind.
- Added a Docker API compatibility preflight before `cac docker create` / `cac docker start`: if the host daemon's minimum API version is newer than the Docker CLI bundled in the image, startup now fails fast with a clear error instead of hanging in readiness.
- Docker mode now defaults to a pinned image tag instead of `latest`, writes that exact image reference into `.env` during setup, and rejects mutable `CAC_DOCKER_IMAGE` values such as `:latest` or tagless refs.
- When the pinned runtime image cannot be pulled but the source tree is available locally, Docker mode now falls back to building that same pinned image tag from the checked-out source instead of silently switching to a mutable ref.

**Validation and docs**

- Added `scripts/validate.sh` as the canonical local validation entrypoint.
- Added CI coverage for the same validation flow and artifact upload.
- Added validation guides in English and Chinese, updated Docker-mode ops docs, and refreshed the real-test guide.
- Added `docker/templates/README.md` and started treating `docker/templates/cherny.*` as version-controlled runtime assets.

## v1.4.4
<sub>2026-03-29</sub>

**Critical fix: Relay killed on every session exit** вҖ” [PR #45](https://github.com/nmhjklnm/cac/pull/45)

- Any `claude -p` one-shot invocation (e.g. from a Feishu/Slack auto-reply pipeline) killed the shared relay process via `_cleanup_all`, breaking all other active sessions with **Connection Refused**. Relay lifecycle changed from session-level to environment-level вҖ” persists across sessions, only stopped by `cac env activate` (switch) or `cac self delete`.
- Added `relay.proxy` file to detect proxy mismatch on startup вҖ” if relay is running with a different proxy, it's automatically restarted.
- `postinstall.js` now patches old wrappers to remove the `trap _cleanup_all EXIT INT TERM` line on upgrade.

**Fix: `cac env check` incomplete** вҖ” [PR #45](https://github.com/nmhjklnm/cac/pull/45)

- Same `pgrep` pipefail bug as the wrapper вҖ” `cac env check` crashed at the concurrent session check, skipping proxy connectivity, exit IP detection, and TUN conflict checks.

**Fix: 10 silent failure points eliminated** вҖ” [PR #45](https://github.com/nmhjklnm/cac/pull/45)

- Systematic audit of all `set -euo pipefail` failures: unguarded `grep` in subshells, `kill` race conditions, empty arrays on bash 3.2, missing `python3` fallbacks, broken symlinks, invalid network interfaces, `openssl` pipeline failures. Each now has explicit error handling instead of silently exiting.

## v1.4.3
<sub>2026-03-27</sub>

**Critical fix: Claude silently fails to launch** вҖ” [commit](https://github.com/nmhjklnm/cac/commit/3258ca9)

- Under `set -euo pipefail`, `pgrep` returns exit code 1 when no `claude` process is running. With `pipefail`, the entire pipeline fails, and `set -e` exits the wrapper before `"$_real" "$@"` executes вҖ” Claude appears to do nothing with no output.
- `postinstall.js` now directly patches existing `~/.cac/bin/claude` on upgrade, so the fix takes effect immediately without requiring the user to run any `cac` command first.
- `cac env ls` now triggers `_ensure_initialized`, enabling automatic wrapper regeneration on version upgrade.

## v1.4.0
<sub>2026-03-27</sub>

**Critical fix: `body.cancel is not a function`** вҖ” [PR #44](https://github.com/nmhjklnm/cac/pull/44)

- dns-guard.js replaced native `fetch` with `node-fetch`, whose `Response.body` is a Node.js Readable stream lacking `ReadableStream.cancel()`. Claude Code runs on Bun and expects native fetch вҖ” every streaming response broke. Now uses URL-checking wrapper instead of replacing the entire fetch implementation.

**Relay hardening** вҖ” [PR #44](https://github.com/nmhjklnm/cac/pull/44)

- **Heartbeat**: Upstream proxy connectivity check every 30s, logs state changes (unreachable вҶ’ recovered)
- **Crash recovery**: `uncaughtException`/`unhandledRejection` handlers prevent single-connection errors from killing the relay process
- **Watchdog**: Wrapper monitors relay process every 10s, auto-restarts on crash вҖ” no more manual session restart
- **Streaming timeout**: Idle timeout 120s вҶ’ 30min, reset on data activity вҖ” long streaming responses no longer killed
- **Fail-closed**: Dead relay вҶ’ HTTPS_PROXY points to dead port вҶ’ connection refused (no IP leak)

**Cleanup** вҖ” [PR #44](https://github.com/nmhjklnm/cac/pull/44)

- Removed deprecated `cac setup` command (auto-bootstrap via `_ensure_initialized` remains)
- Updated stale error messages referencing `cac setup`

**Security hardening** вҖ” [Issue #42](https://github.com/nmhjklnm/cac/issues/42)

Based on community reverse engineering of Claude Code 2.1.83 (11.3MB JS bundle analysis by [@mo-xiaoxi](https://github.com/mo-xiaoxi)):

- **Telemetry modes**: Three modes вҖ” `off` (no intervention, for pure API Key users) / `conservative` (new default, blocks non-essential traffic) / `aggressive` (12-layer full block)
- **Block `CLAUDE_CODE_ENHANCED_TELEMETRY_BETA`**: Preemptively prevent enhanced telemetry activation via future feature flags
- **Persist `firstStartTime` per environment**: Each environment maintains consistent "device age", avoiding the "perpetual new device" detection pattern
- **Concurrent session warning**: Warns when claude process count exceeds threshold (default 10, configurable via `~/.cac/settings.json`)
- **`LANG` auto-adaptation**: Locale now auto-detected from exit IP country code (e.g. JPвҶ’`ja_JP.UTF-8`, SGвҶ’`en_SG.UTF-8`)
- **`cac env check` enhancements**: Runtime fingerprint verification, IPv6 leak detection, TZ/IP consistency check, residual telemetry file detection

**Config inheritance** вҖ” [Issue #43](https://github.com/nmhjklnm/cac/issues/43)

- **`--clone` flag**: `cac env create work --clone` inherits config from `~/.claude/` (commands, hooks, skills, plugins via symlink; CLAUDE.md via symlink; settings.json via layered merge)
- **`--clone <env>`**: Clone from another cac environment instead of host
- **`--no-link`**: Copy files instead of symlink for independent customization
- **CAC Meta Prompt**: `--clone` with symlink skips CLAUDE.md injection; `--clone --no-link` appends cac instructions to copied CLAUDE.md
- **`settings.override.json`**: Environment-specific settings layered on top of shared base config, auto-merged on each startup
- **`~/.cac/settings.json`**: New cac-level config file for global preferences (max_sessions, telemetry_mode)

**Infrastructure**

- **Wrapper auto-upgrade**: `CAC_WRAPPER_VER` stamp detects version mismatch вҶ’ wrapper auto-regenerated on next command
- **Wrapper optimizations**: Settings merge skipped when inputs unchanged (mtime check), session count reads file instead of spawning python3
- **macOS fixes**: `pgrep -cf` unavailable on macOS вҶ’ `pgrep -f | wc -l`; `pgrep -x` for exact claude binary match (avoids MCP subprocess false positives)
- **`--clone` arg parsing fix**: `--no-link` no longer consumed as clone source name

## v1.0.0

**Initial architecture** вҖ” [PR #20](https://github.com/nmhjklnm/cac/pull/20)

Rewrite from proxy management script to uv-style Claude Code manager:

- **Version management**: `cac claude install/ls/pin/uninstall`
- **Environment isolation**: `cac env create/ls/rm` with per-environment `CLAUDE_CONFIG_DIR`
- **Node.js fingerprint hook**: Intercepts `os.hostname()`, `os.networkInterfaces()`, `child_process.execSync('wmic/ioreg')` вҖ” [PR #5](https://github.com/nmhjklnm/cac/pull/5)

- **Health check bypass**: In-process Node.js interception, no `/etc/hosts` or root needed
- **Local relay**: Auto-bypasses TUN interfaces via loopback forwarding
- **mTLS**: Self-signed CA + per-environment client certificates
- **Docker mode**: sing-box TUN network isolation + cac identity protection вҖ” [PR #8](https://github.com/nmhjklnm/cac/pull/8)
